## 2. Technical Scope & Workspace Principles
*   **Application**: Custom smart servo control board designed to fit completely inside a gutted TowerPro MG996R casing for a high-performance bipedal robot joint.
*   **Architecture**: Local-first, real-time PID feedback loop for dynamic, compliant, and "squishy" torque/impedance tracking at a target rate of 1kHz.
*   **Power Rail Architecture**: Shared single regulated 5V power bus for both the logic circuit and the DC motor. No onboard LDO regulator to save space. 
*   **Noise Isolation Strategy**: Strict hardware filtering due to shared 5V bus. Utilizes an RC low-pass filter (10Ω series resistor + 10µF ceramic capacitor) on the MCU supply line to suppress high-frequency brush noise and PWM transients. 
*   **Grounding Rules**: Star grounding topology to isolate high-current motor return paths from sensitive analog feedback circuits.

## 4. Hardware, Circuit, & Component Context
To prevent hallucination, the system must strictly adhere to the tested parameters of the selected Bill of Materials:

### Core Mechanical & Electrical Donor Specs
*   **Servo Model**: TowerPro MG996R (or standard metal-gear clone).
*   **Maximum Current Rating (Stall)**: **2.5A to 3.0A peak** during heavy dynamic walking or sudden joint stalls.
*   **Feedback Mechanism**: Stock internal 3-pin potentiometer retained for absolute joint position. Wired directly to MCU VCC (5V), GND, and an ADC input pin.

### Finalized Component Specs
*   **Motor Driver**: BDR612H (SOP-8 package, up to 3A peak stall handling capability).
*   **Transient Protection**: SMAJ5.0A Unidirectional TVS Diode (SMA/DO-214AC package). Cathode wired to 5V, Anode to GND. Clamps back-EMF spikes generated by physical leg impacts or abrupt motor reversals.
*   **Capacitance**: 
    *   1x 100µF  miniature electrolytic bulk capacitor across the power entry point to smooth voltage sags.
    *   1x 10µF ceramic capacitor (0805) for the MCU RC filter.
    *   2x 0.1µF (100nF) ceramic capacitors (1206) placed directly at the VCC/GND pins of the MCU and the driver chip.
*   **Filter Passives**: 1x 10Ω resistor (0805) in series with the MCU power line; 1x 4.7kΩ pull-up resistor on the half-duplex UART communication line.

### Current Sensing Architecture (Low-Side Differential)
Current sensing is implemented on the low-side of the H-bridge driver. Due to the tight space constraints, the sensing must be purely analog without external amplifier ICs. The choice of MCU alters this architecture entirely:

*   **CH32V003**: Uses its internal Programmable Gain Amplifier (up to 32x) to scale a minute 125mV drop across a **50mΩ** current shunt resistor. Programmed via Raspberry Pi Pico running PicoRVD firmware over a 1-wire interface with a 1kΩ pull-up on the line.

## 5. Microcontroller Firmware Responsibilities
The selected MCU (regardless of architecture) is tasked with executing a tight, multi-threaded or low-latency main loop performing the following concrete actions:

1.  **Position Feedback Tracking**: Constantly sample the analog voltage from the servo's internal potentiometer wiper via an ADC pin to track absolute joint angle.
2.  **Torque / Current Monitoring**: Sample the voltage drop across the low-side shunt resistor via the ADC to monitor instantaneous current draw (representing load/torque).
3.  **Real-Time 1kHz PID Execution**: Compute a Proportional-Integral-Derivative algorithm comparing target joint position vs. actual position, heavily modulated by the current readings to introduce "squishiness" (impedance compliance under high external loads).
4.  **Motor Driver Command Generation**: Output high-frequency PWM signals to the BDR612H driver to control direction and speed based on PID computations.
5.  **Bus Communication**: Listen and respond to position targets or configuration parameters sent down the robot's leg via a single-wire Half-Duplex UART serial communication line.

## 7. The Software Architecture (Phase 1)
| Firmware Module | Primary Goal | The "Gotcha" to watch out for |
| :--- | :--- | :--- |
| **1. PWM & Driver** | Generate complementary 20kHz PWM on two pins to drive the BDR612H forward and reverse. | Setting "Deadtime." If you switch directions instantly, you'll shoot a massive current spike and trip the power supply. |
| **2. ADC & DMA** | Read the Potentiometer (Position) and the Shunt (Current) asynchronously. | You cannot use blocking `analogRead()` calls. We need to set up Direct Memory Access (DMA) so the ADC reads in the background without freezing your 1kHz loop. |
| **3. The PID Loop** | The actual math block. Compare target position to real position, modulate by current. | Integral Windup. If you hold the servo back, the "I" term will build up forever and violently snap the motor when you let go. |
| **4. UART Comms** | Listen for target angles from your main controller via the single wire. | Managing the TX/RX buffer seamlessly without missing incoming bytes while calculating PID. |
